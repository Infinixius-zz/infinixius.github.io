Project Contributing

In order to keep issues and merge requests clean, there are a set of requirements you must meet before being accepted.

Did you find a bug?

  • If the bug is a security vulnerability, do NOT report it. Instead, report it privately to me.
  • Ensure the bug is not already reported, by searching on the Github repository for issues.
  • Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

If your issue follows the above criteria, open a new issue.

Did you write a patch that fixes a bug?

  • Ensure the merge request description clearly describes the problem and solution. Include the relevant issue number if applicable.
  • If the patch solves a security vulnerability, do NOT open the merge request. Instead, report the fix privately.
  • If your patch is cosmetic in nature and does not add anything substantial to the stability, functionality, or testability of the project, it probably won't be accepted.
  • Ensure that your patch doesn't break any existing code.

If your patch follows the above criteria, open a new pull request.

Do you intend to add a new feature or change an existing one?

  • Ensure you have collected positive feedback about the change. You can suggest the new feature/change on Reddit to recieve feedback.
  • Ensure the change doesn't break any existing code.
  • Ensure the feature actually serves a purpose.
